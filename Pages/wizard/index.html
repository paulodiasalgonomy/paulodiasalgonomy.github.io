apiKey

Data Type: String | Unique key that identifies the site. Each partner has a unique API key that separates that partner’s data and traffic from other partners.

apiClientKey

Data Type: String | Unique key specific to each API implementation. Identifies the specific application and channel.

Required Parameters

userId

(required when user is logged)

Data Type: string | Unique string to identify each user. All user behavior will be stored using this key. It is case-sensitive and should be the same UserId sent to {rr} in other applications.

sessionId (required )

Data Type: string | Identifies one visit, or use of a site or application by a user. Used to determine “scope” when building behavioral models.

rcs (required )

Algonomy Encrypted Cookies. It must be sent as received in the previous API response.

Optional Parameters

ipor - Shopper IP address

Data Type: string | When using location-based strategies, use this parameter to relay the IP address to RichRelevance servers. Only use this parameter in a server to server implementation, when the API call is proxied or not originated by a browser.

IS REQUIRED TO CREATE GEOLOCALIZED CAMPAINGS

pref - Referral URL

Data Type: string | The address from where the current user landed before seeing the current page. Use this value to enable Conditional Placements. Value must be URL-encoded.

IS REQUIRED TO CREATE REFERRAL CAMPAINGS
